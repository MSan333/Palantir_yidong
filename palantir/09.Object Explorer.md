**Object Explorer** 是 Palantir Foundry 中用于**探索和分析本体（Ontology）对象**的首选点选式工具 [1, 2]。它为用户提供了一个安全、可扩展且无需编写代码的界面，用于快速回答业务问题并进行深度数据调查 [1-3]。

以下是根据来源对 Object Explorer 核心功能和操作流程的详细介绍：

### 1. 搜索与探索 (Search & Exploration)
*   **对象全景视图：** 进入应用后，用户可以看到其拥有权限的所有对象类型，这些对象通常按照在 Ontology Manager 中定义的业务组（如“主题公园餐饮”）进行分类 [1]。
*   **动态过滤与对象集（Object Sets）：** 用户通过搜索栏或点击图表中的值来应用过滤器 [4]。这一过程会产生一个**“对象集（Object Set）”**，它不是静态的，而是一个动态的集合，类似于 SQL 查询，会随着底层数据的更新而实时变化 [4, 5]。
*   **查看对象详情：** 点击单个对象会打开其“对象视图（Object View）”，这是该对象实例的视觉化呈现，其界面可以通过 Workshop 模块进行深度自定义 [5]。

### 2. 视觉化分析与自定义布局 (Visual Analysis & Layouts)
*   **内置与自定义图表：** 探索页面预配置了一系列图表（如直方图），展示对象属性的分布情况 [4]。用户可以添加新图表，并利用“配置（Cog）”图标修改展示方式，例如将分布图转化为**统计表（Stats Table）**或按另一属性进行分层展示 [6, 7]。
*   **保存布局（Layouts）：** 用户可以自定义列的显示或隐藏，并调整图表位置 [8]。这些自定义配置可以保存为“布局（Layout）”，并设定为仅限自己可见或作为组织内所有用户的默认视图 [8, 9]。

### 3. 关联对象钻取与比较 (Linked Objects & Comparison)
*   **跨对象钻取：** Object Explorer 允许用户利用本体中的链接进行“钻取（Pivot）” [10]。例如，从一组大额餐饮订单直接跳转到关联的“客户”对象，以分析高消费人群的地理分布（如本地或国际客户） [10, 11]。
*   **对象集比较：** 用户可以创建多个对象集并进行对比分析 [12]。例如，将“高消费客户”与“低消费客户”放在一起，通过图表对比发现他们在游乐项目偏好上的显著差异 [12-14]。

### 4. 结果保存与业务触发 (Saving & Operationalizing)
*   **探索（Exploration）与列表（List）：** 
    *   **探索**是动态的，保存的是筛选标准，结果会随数据变化而更新 [11]。
    *   **列表**是静态的，像一张快照，保存的是特定时刻的一组对象 [15]。
*   **自动监控：** 选定的对象集或探索可以通过 **Automate** 进行监控 [15]。当新的对象符合特定标准（如单笔消费超过 $200）时，可以触发自动化操作，例如发送感谢邮件或优惠券 [15]。
*   **集成操作（Actions）：** Object Explorer 旨在融入操作流，用户可以直接对筛选出的对象应用 **Action**（操作），从而将分析洞察转化为实际的业务决策 [3, 16]。

### 5. 跨应用协同
Object Explorer 并不是分析的终点。用户可以将筛选出的结果快速**导出或在其他 Foundry 应用中打开**，包括在 Quiver 中进行更复杂的分析、在 Contour 中作为数据集处理，或在数据谱系（Data Lineage）中查看其底层数据来源 [14, 17]。

总的来说，Object Explorer 是 Foundry 的**“数字孪生浏览器”**，它让非技术用户也能通过直观的操作，在复杂的企业级本体数据中发现模式并执行操作 [2, 18]。
