**Code Workspaces** 是 Palantir Foundry 提供的一款集成开发环境应用，允许用户直接在平台内使用 **Jupyter Notebooks** 和 **RStudio** 等第三方 IDE [1]。它的核心价值在于让数据科学家和分析师能够使用自己熟悉的工具对高质量数据进行分析，同时保持完整的**数据溯源（Data Provenance）** [1]。

以下是关于 Code Workspaces 的详细介绍：

### 1. 核心功能与 IDE 选择
*   **第三方 IDE 集成：** 用户可以根据需求创建基于 **Jupyter Notebook** 或 **RStudio** 的工作空间 [1]。
*   **计算资源配置：** 在创建时，用户可以自定义高级设置，包括**计算资源**的分配以及空置超时（Timeout）时间 [1]。
*   **版本控制：** Code Workspaces 的所有工作都由底层的 **Code Repository（代码仓库）** 支持，这意味着它具备与 Code Repos 相同的版本控制能力，支持分支管理和代码同步 [2]。

### 2. 用户界面（UI）构成
工作空间的界面布局通常分为几个关键区域：
*   **中间区域：** 展示主要的 IDE 界面（如 Jupyter Notebook 单元格） [2]。
*   **左侧面板：** 包含 **Foundry 特有的交互功能**，如数据集成和资源管理 [2]。
*   **顶部栏：** 提供切换分支、同步更改（Sync Changes）等功能，点击同步会向底层的仓库提交（Commit）更改 [2]。

### 3. 数据交互流（End-to-End Workflow）
在 Code Workspaces 中，数据的导入和导出通过代码片段（Snippets）与平台无缝连接：

*   **导入数据（Data Ingress）：**
    *   用户在数据面板中选择现有数据集并为其设置**别名（Alias）** [2]。
    *   系统会自动生成一段代码片段（例如将数据导入为 **Pandas DataFrame**），用户将其粘贴并运行即可获取数据 [3]。
*   **处理数据：**
    *   用户可以利用安装的第三方库（如 Pandas、Matplotlib）进行数据清洗、聚合和可视化绘图 [3, 4]。
*   **导出数据（Data Egress）：**
    *   用户可以在数据面板中选择“将数据写入新数据集（Write data to a new dataset）” [4]。
    *   指定变量名和目标数据集名称后，系统生成代码片段来执行写入操作 [4, 5]。
    *   **同步与构建：** 运行写入代码并点击“同步并运行（Sync and Run）”后，系统会创建转换逻辑并触发构建，用户可以通过“查看检查（View Checks）”监控构建进度 [5]。

### 4. 与其他工具的关系
*   **底层支撑：** 虽然前端是交互式的 Notebook，但其本质仍是受版本控制的代码，这种机制确保了分析过程的可重复性和安全性 [2, 5]。
*   **产出物：** 工作空间不仅可以生成分析报告和图表，还可以产出新的数据集，供 Foundry 平台内的其他应用（如 Workshop 或 Contour）直接使用 [3, 6]。

通过 Code Workspaces，Foundry 将传统数据科学的灵活性与企业级的数据治理结构结合在了一起。
